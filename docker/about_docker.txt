Dockerについて

・アプリの実行環境をラップしてくれる
 - ホストがライブラリやミドルウェアで汚れない
 - ネットには便利なコンテナがゴロゴロ

・VMと比べて軽い
 - 軽い

・container
 - オブジェクト指向でいう[インスタンス]
 - imageから作られる実際の動作環境　
 - 原則1コンテナ1アプリ
 - コンテナ起動 
  - docker run   -> imageから作成
  - docker start -> すでに作成済みのコンテナを起動する

・image
 - オブジェクト指向でいう[クラス]
 - テキストファイル(Dockerfile)からビルドされる
 - すでにほとんどの実行環境が配布されている
 -

 Docker composeについて
 - yamlファイル１つで複数コンテナを定義
 - docker-compose.yml(各種設定を行うファイル ポートフォワーディング等)
